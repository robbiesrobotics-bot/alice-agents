# v1.5.14

Status: approved

## Summary
- Made Hermes installs profile-aware so A.L.I.C.E. writes skills, memories, config checks, and installer status into the active Hermes profile.
- Added shared Hermes path helpers covering default profile, named profiles, and explicit `HERMES_HOME` runs.
- Updated README compatibility docs and added regression tests for profile-aware path behavior.

## Changes

- **Profile-aware Hermes paths** — new `lib/hermes-paths.mjs` centralizes Hermes path resolution. `HERMES_HOME` is authoritative when present; `HERMES_PROFILE` / `HERMES_PROFILE_NAME` route named profiles to `~/.hermes/profiles/<profile>`; `default` remains `~/.hermes`.
- **Installer writes stay inside the active profile** — `lib/hermes-installer.mjs` now installs A.L.I.C.E. skills, memory directories, `.alice-config.json`, and model/config checks under the active Hermes profile instead of the root Hermes home.
- **Hermes agent manager uses active profile skills** — managed Hermes specialist skills now live under the active profile `skills/` directory, while binary discovery still checks profile-local, base Hermes, user-local, and PATH locations.
- **Runtime detection aligned with profiles** — installer and runtime detection now check the active Hermes profile config path, avoiding false negatives and accidental root-profile installs.
- **Docs and tests** — README compatibility text documents named profile paths, and `test/hermes-paths.test.mjs` covers default, named-profile, explicit `HERMES_HOME`, and `default` profile behavior.

## Compatibility

- **Default Hermes users** — no path change; installs still land in `~/.hermes/skills/alice/` and `~/.hermes/memories/`.
- **Named Hermes profiles** — installs now land in `~/.hermes/profiles/<profile>/skills/alice/`, with status/config/memory files kept profile-local.
- **Existing installs** — no automatic migration is performed. Re-running `npx @robbiesrobotics/alice-agents --runtime hermes --update` from the desired Hermes profile refreshes that profile's skill tree.

## Announcement

A.L.I.C.E. v1.5.14 fixes Hermes profile support for real-world multi-profile installs. Users running Hermes with named profiles now get A.L.I.C.E. installed exactly where that profile expects it, rather than leaking skills and memory into the root `~/.hermes` tree. Default-profile users are unchanged.

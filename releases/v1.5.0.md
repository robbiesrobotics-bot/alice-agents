# v1.5.0

Status: approved

## Summary
- Hardened model-agnosticism across Mission Control web app — removed all hardcoded model names, provider fallbacks, and constrained enums that leaked back after v1.4.2
- Agent creation, LLM service, embedding service, agent service, telemetry, and session mocks now pass through whatever model/provider is selected in OpenClaw/NemoClaw

## Announcement
`v1.5.0` finishes the model-agnostic cleanup we started in v1.4.2. Mission Control no longer assumes Claude, Gemini, or GPT anywhere in its runtime paths — your configured model flows through cleanly end-to-end. Agent creation accepts any model string, the LLM service delegates to the gateway without injecting defaults, and the embedding service now reads model names from `EMBEDDING_MODEL` env var.

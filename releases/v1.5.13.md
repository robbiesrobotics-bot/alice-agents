# v1.5.13

Status: approved

## Summary
- Retired the Mission Control cloud bridge — Alice Hub is now the web product surface, so the alice-agents installer no longer carries telemetry/ingest plumbing the runtime doesn't need
- Hardened the starter installer's tier handling — Athena's allow-list no longer dangles a reference to a Pro-only specialist on Starter installs, and a defensive registry filter drops any future dangling allowAgents entries with a logged warning instead of silently breaking delegation
- Manifest is self-healing: legacy `missionControl` blocks are stripped on first write, and `packageVersion` now refreshes on every install instead of pinning to whatever wrote the manifest first

## Changes

- **Mission Control bridge fully removed** — deleted `lib/mission-control.mjs`, `bin/alice-cloud.cjs`, and `templates/mission-control-bridge/`; stripped MC blocks from `lib/installer.mjs`, `lib/manifest.mjs`, `lib/doctor.mjs`, `lib/prompter.mjs`; dropped `--cloud`, `--no-cloud`, and the `--cloud-*` configuration flags from `bin/alice-install.mjs`. License validation against `getalice.av3.ai/api/license/validate` is unaffected — only the cloud telemetry/ingest path was removed.
- **Defensive registry filter** — `lib/agent-registry.mjs` now drops any `allowAgents` entries pointing at agents not present in the resolved tier roster, with a `[alice-agents] <agent>: dropping unknown allowAgents references […]` warning. Fixes the silent-delegation-failure case where Starter Athena referenced `nadia` (Pro-only).
- **Manifest writer cleanup** — `missionControl` field is destructured off the existing manifest on every write so legacy installs migrate themselves; `packageVersion` always reflects the installer that wrote the manifest (was previously sticky on the first write).
- **Tier prompt counts corrected** — Starter shows "14 agents", Pro shows "22 more agents, 36 total" (was 10 / 18).
- **Symphony fan-out on Athena's coding team** — Athena's AGENTS.md gained a Symphony Fan-out section describing the project workspace + clone fan-out loop; Sasha, Dylan, Morgan, Priya, Felix, Quinn, and Devon templates gained the "Working in a Project" addendum that documents the `project_workspace` and `clone_self` tools they see when dispatched into a project. Templates also set `subagents.cloneSelf.max = 4` for those nine specialists and grant `project_workspace` in `tools.alsoAllow` for all 14 starter + 22 pro agents.
- **Test suite expanded** — added regression coverage for the new dangling-ref filter (Starter drops `nadia`; Pro keeps `nadia`) so the dependency between starter and pro templates can't silently rot.

## Compatibility

- **Local installs** — `npx @robbiesrobotics/alice-agents` (interactive) and `--update` (non-interactive) are unchanged for the user. Users who had previously enabled the cloud bridge will see the `missionControl` block disappear from `~/.alice/.alice-manifest.json` (or `~/.openclaw/openclaw.json`) on next install — that's the migration. No data is lost; the local cloud config files at `~/.openclaw/.alice-mission-control.json` and `~/.openclaw/alice-cloud.json` become orphans that can be deleted manually.
- **Existing tier=pro installs** — defensive filter is a no-op when the full Pro roster is resolved, so delegations to `nadia`, `clara`, `sloane`, etc. continue to work.
- **CI / automation** — the removed `--cloud`, `--no-cloud`, `--cloud-token`, `--cloud-dashboard-url`, `--cloud-ingest-url`, `--cloud-team-*` flags now no-op (treated as unknown args). Update any pipelines that passed them.

## Announcement

A.L.I.C.E. v1.5.13 retires the Mission Control cloud bridge. The alice-agents npm package is now strictly an installer for the local agent team — telemetry, fleet visibility, and dashboard lives in Alice Hub, our runtime-native web product. The installer also stops shipping a dangling reference to Nadia in Starter Athena's allow-list and self-heals legacy `missionControl` manifest fields on next write.

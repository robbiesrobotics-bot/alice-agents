# v1.4.7

Status: approved

## Summary
- Mission Control Cloud installs now persist team identity in local config and send it through the bundled bridge
- Hosted runtime registration, heartbeats, and session events now carry `teamId`, letting Mission Control and admin.av3.ai attach runtime activity to the right shared-team records
- This release lines up the installer with the new shared Supabase control-plane work so production dashboards can stay team-scoped

## Announcement
`v1.4.7` turns the A.L.I.C.E. cloud bridge into a team-aware runtime. New and upgraded installs now send `teamId` alongside node and session telemetry, which means Mission Control and `admin.av3.ai` can finally connect hosted activity, fleet state, and incidents to the right shared team without falling back to local-machine assumptions.

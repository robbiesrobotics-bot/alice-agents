## Analysis

### 📈 Wins

1. **1,441 downloads in Week 1 is a strong cold start.** For a brand-new developer package with zero advertising, breaking 1,000 downloads in the first week is a signal that the audience exists and is finding the package organically.

2. **The Saturday spike (488 downloads) correlates with v1.5.0 launch.** This suggests that version releases drive download events — likely automated re-installs, update checks, or CI pipelines resolving to latest. This correlation should be tracked closely in future weeks.

3. **Release velocity is aggressive and deliberate.** 29 versions in 7 days (avg ~4.1/day) indicates tight iteration loops. The build is guarded by a prepublishOnly test suite (103/103 passing as of 2026-03-21), so this isn't thrash — it's intentional rapid shipping.

4. **Package structure is now mature.** Latest version (v1.5.1) includes real Starter + Pro registries, Mission Control cloud setup, hosted telemetry with teamId, and node heartbeat reporting. The product is no longer scaffolding.

### ⚠️ Anomalies & Watch Items

1. **Zero-download days (Mar 17–19).** Three consecutive zero-download days following launch is abnormal. Possible explanations: npm CDN cache warming delay, no automated installs polling those days, or the early install spike was purely manual. Worth monitoring in Week 2 — if similar gaps appear post-release, this may indicate no CI/CD pipelines are pulling the package yet.

2. **GitHub repo is private.** `api.github.com/repos/robbiesrobotics/alice-agents` returns 404. Stars, forks, and issue counts are unavailable. This is expected if the repo hasn't gone public yet. Once public, these metrics become critical trust signals. (Note: This was also the case as of 2026-03-19 per Sloane's sales signal report.)

3. **Download/release mismatch.** Mar 17–19 had 14 version publishes but 0 downloads logged. This may be a timing artifact in the npm stats API (stats can lag 24–48h), or it indicates the audience is not yet monitoring the registry for updates.

### 🔍 Week 2 Hypotheses to Track

- Does each major version bump (v1.x.0) produce a download spike?
- Does going public on GitHub correlate with a download acceleration?
- What is the baseline weekly download rate now that the launch spike has settled?

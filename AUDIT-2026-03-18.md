## Critical Issues (Must Fix)

### 🔴 1. Stripe is in TEST MODE
**Where:** getalice.av3.ai — POST /api/stripe/checkout returns `cs_test_` prefixed sessions.
**Impact:** No real payments can be processed. This is a launch blocker.
**Fix:** Set `STRIPE_SECRET_KEY` and `NEXT_PUBLIC_STRIPE_PUBLISHABLE_KEY` to live keys in Vercel environment variables for getalice.av3.ai.

### 🔴 2. alice.av3.ai is completely unreachable (SSL/TLS failure)
**Where:** alice.av3.ai — TLS handshake fails, no certificate presented.
**Impact:** Mission Control is dead. Any user directed here (including from docs/landing) hits a dead end. Major credibility issue.
**Fix:** Check Vercel project for alice.av3.ai — confirm deployment exists and custom domain is properly configured with SSL provisioned. Re-add domain if deleted.

### 🔴 3. License validation API returning 503
**Where:** GET /api/license/validate?key=X → HTTP 503 "License validation unavailable"
**Impact:** Pro license keys cannot be validated server-side. The CLI falls back gracefully (per v1.3.1 changelog), but any integration relying on server validation is broken.
**Fix:** Check the license validation service/database connection. Likely a Supabase or env var issue in the getalice.av3.ai deployment.

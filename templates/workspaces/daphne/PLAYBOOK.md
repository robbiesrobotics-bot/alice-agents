## Patterns

### Agent Workspace Provisioning
When creating workspace files for A.L.I.C.E. agents, the standard file set is:
1. **SOUL.md** — personality, boundaries, vibe (CRITICAL - do first)
2. **AGENTS.md** — operating instructions with learning boot sequence (CRITICAL - do second)
3. **IDENTITY.md** — name, emoji, one-liner
4. **USER.md** — info about Rob (mostly boilerplate)
5. **TOOLS.md** — local notes (mostly scaffold)
6. **PLAYBOOK.md** — evolved expertise (scaffold)
7. **LEARNINGS.md** — task reflections (scaffold)
8. **FEEDBACK.md** — user feedback log (scaffold)

Work **agent-by-agent**, not file-type-by-file-type. Complete one agent fully before starting the next.

### Two AGENTS.md Templates
- **Coding agents** (dylan, darius, avery, isaac, alex, felix, caleb, quinn, devon): Include `claude_code` skill reference
- **Non-coding agents** (everyone else): Use "available tools" phrasing instead

### Boot Sequence (required in every AGENTS.md)
```
1. Read SOUL.md
2. Read PLAYBOOK.md (if exists)
3. Read FEEDBACK.md (if exists)
4. Skim LEARNINGS.md (last 10 entries)
5. Read the task
6. Check memory/
```

### Post-Task Reflection (required in every AGENTS.md)
Include the structured reflection template pointing to LEARNINGS.md.

## Anti-Patterns

- Don't attempt 100+ file writes in a single session without flagging the time cost
- Don't assume gateway won't restart mid-task — prioritize critical files first
- Don't use the same generic SOUL.md for all agents — customize domain values and vibe
